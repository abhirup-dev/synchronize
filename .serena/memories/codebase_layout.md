# Codebase Layout

## Root entrypoints

- `bin/synchronize` -> CLI main from `src/cli/`.
- `bin/synchronize-mcp` -> MCP stdio adapter from `src/mcp/`.
- `src/daemon.ts` -> Bun daemon and HTTP route owner.
- `src/cli.ts`, `src/mcp.ts`, `src/api.ts` are compatibility shims; preserve their exports when refactoring.

## Source tree

- `src/api/`: typed REST facade and DTOs. Modules currently include `status`, `peers`, `inbox`, `events`, `groups`, `media`, and `agent-sessions`.
- `src/cli/`: argv parsing, command dispatch, identity helpers, warnings, and render helpers. Commands include `hook` and `launch` in addition to messaging/group/media/status commands.
- `src/mcp/`: stdio server, lifecycle, adapter state, notifications, Claude callback subscription, Codex polling notifier, and tool registrars under `src/mcp/tools/`.
- `src/launch/`: daemon-managed session launch layer. `build.ts` builds agent argv/env, `service.ts` owns launch intent and pending reconciliation, `backend.ts` implements the AOE backend.
- `src/db.ts`, `src/fs.ts`, `src/http.ts`, `src/paths.ts`, `src/constants.ts`, `src/provenance.ts`: shared infrastructure.
- `web/src/`: React UI, data adapters for daemon/mock state, room/message/sidebar/thread components, and keyboard/scroll hooks.
- `extensions/pi-synchronize/`: Pi coding-agent extension with its own daemon client, subscription, delivery, identity, and logging code.

## Supporting folders

- `tests/`: Bun tests for daemon/API/MCP/messaging/presence/launch/AOE/backend behavior.
- `scripts/`: host config scripts, AOE teardown, resilient MCP command generation, and the Python real-agent integration harness under `scripts/integration-aoe/`.
- `docs/`: agent docs, plans, ADRs when present, and operational design notes.
- `.claude/`: Claude settings, handoffs, skills, and local worktrees; Graphify should index useful docs/skills but ignore `.claude/worktrees/` and local launch config.
- `.serena/`: Serena project config and memories.
- `graphify-out/`: generated knowledge graph output; ignored by git and regenerated by Graphify.

## Files to treat carefully

- `src/daemon.ts`: still a large route/repository/fanout owner; route changes can affect CLI, MCP, web, and Pi.
- `src/client.ts` and `src/api/*.ts`: shared contract surface for all adapters.
- `src/mcp/lifecycle.ts` and `src/mcp/tools/register.ts`: peer identity, heartbeat, and launch-bound startup behavior.
- `scripts/claude-hooks-config.ts` and `src/cli/commands/hook.ts`: Claude hook installation and session/activity ingestion.
- `src/launch/*`: daemon-managed AOE session launch behavior.
- `Makefile`, `bin/*`, and install scripts: host integration surface.
